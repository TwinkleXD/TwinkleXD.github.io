React官方文档学习

环境

前提准备

  • Node.js (>=8.10)
  • npm (>=5.6)

创建项目

核心概念

JSX

JavaScript 的语法扩展

  1. 可以直接在js里写标签
  2. 标签内可以在大括号中放任何有效的 JS 表达式
  3. 标签属性可以通过引号或者大括号指定
  4. JSX 直接使用是安全的（React DOM 在渲染输入内容之前会转义成字符串，有效地防止XSS（cross-site-scripting, 跨站脚本）攻击）
  5. React 元素：描述显示内容，是普通对象，不可变
    1. Babel 把 JSX 转译成React.createElement()函数调用
    2. React.createElement()预先执行检查，创建对象

  • JSX –> Babel 转译 –> 实际创建元素

    《React 官方文档》

元素渲染

  1. React DOM 负责更新浏览器 DOM 与 React 保持一致
  2. ReactDOM.render()将 React 元素渲染到根 DOM 节点中
  3. 当前更新 UI 的方式：创建全新元素传入ReactDOM.render()
  4. React DOM 会将元素和它的子元素与它们之前的状态进行比较，只更新需要更新的部分

组件 & Props

  1. 函数组件与 class 组件，以下为二者等效

  2. 当 React 元素为用户自定义组件时，会将 JSX 所接收的属性（attributes）以及子组件（children）转换为单个对象传递给组件，即props

  3. React 会将以小写字母开头的组件视为原生 DOM 标签，所以自定义组件名称必须以大写字母开头

  4. 组合、提取组件其实都是将自定义组件作为标签复用、嵌套，需要注意 props 的传递

  5. Props 具有只读性，组件不能修改自身的 props

    • 纯函数：不更改入参，多次调用相同入参返回值相同

      所有 React 组件都必须像纯函数一样保护 props 不被更改

State & 生命周期

参考：ReactJs基础知识04–state&&生命周期

state

state 与 props 类似 ，但是 state 是私有的，完全受控于当前组件

  1. 在 class 组件中添加构造函数，初始化this.state
  2. class 组件应该始终使用props参数来调用父类的构造函数，即super(props)，es6中，关键字 super 代表父类实例
  3. state的使用：
    1. 不要直接修改state，使用setState()
    2. 异步更新时，让setState()接收函数而不是对象
    3. state 的更新会被合并，setState 并不会立即改变 state 的值，而是将其放到一个任务队列里，最终将多个 setState 合并，一次性更新页面

生命周期

  1. 挂载：组件被渲染到 DOM 中
  2. 生命周期方法：组件挂载或者卸载（ DOM 中的组件被删除）时执行的方法
  3. 具体方法：
    1. componentWillMount：在渲染前调用,在客户端也在服务端
    2. componentDidMount：在第一次渲染后调用，只在客户端
      • 之后组件已经生成了对应的 DOM 结构，可以通过this.getDOMNode()来进行访问
    3. componentWillReceiveProps：在组件接收到一个新的 prop(更新后) 时被调用
    4. shouldComponentUpdate：返回一个布尔值。在组件接收到新的 props 或者 state 时被调用
    5. componentWillUpdate：在组件接收到新的 props 或者 state 但还没有 render 时被调用
    6. componentDidUpdate：在组件完成更新后立即调用
    7. componentWillUnmount：在组件从 DOM 中移除的时候立刻被调用

单项数据流

  1. 任何的 state 总是所属于特定的组件，而且从该 state 派生的任何数据或UI只能影响树中“低于”它们的组件
  2. 在 React 应用中，组件是有状态组件还是无状态组件属于组件实现的细节，它可能会随着时间的推移而改变。你可以在有状态的组件中使用无状态的组件，反之亦然

事件处理

  1. 命名采用驼峰式
  2. JSX 直接传入函数作为事件处理函数，而不是字符串
  3. 不能通过return false阻止默认行为，要显式的用preventDefault
  4. JS 中 class 的方法默认不会绑定 this ，需要手动绑定
    1. 使用 bind 绑定 this:
    2. class filed (实验性语法)
    3. 回调中使用箭头函数 (每次渲染时都会创建不同的回调函数，作为 props 传入子组件时，可能会额外重新渲染，不推荐)
  5. 向事件处理程序传递参数

条件渲染

  1. 函数组件根据条件判断渲染返回的内容
  2. 根据元素变量进行条件判断
  3. 运算符判断: &&、三元运算符
  4. 阻止组件渲染：render方法返回null，不会进行任何渲染，也不会影响组件的生命周期

列表 & Key

  1. 数组转化成元素列表: map() 遍历数组元素插入<li>标签得到新数组，将新数组插入到<ul>标签中，渲染进 DOM
  2. key: 赋予数组中每个元素唯一标识，帮 React 识别哪些元素改变了，比如被添加或删除
  3. 提取组件时应该把key元素保留在提取的组件中
    • 在map()方法中的元素需要设置 key 属性
  4. 数组元素中使用的 key 在其兄弟节点之间应该是独一无二的
  5. key 会传递信息给 React ，但不会传递给组件

表单

  1. 受控组件: 元素输入的值始终由 React 的 state 驱动，state更新，render函数都会执行,因此显示的值将随着输入改变而更新，实现了双向数据绑定
  2. <textarea>、<select>与 HTML 中有所不同
  3. <input>value只读，是非受控组件
  4. 处理多个输入：给每个元素添加 name 属性，并让处理函数根据event.target.name的值选择要执行的操作
  5. 受控组件指定 value 会阻止用户更改输入，value 设置为空仍可以编辑

状态提升

多个组件需要反映相同的变化数据，将共享状态提升到最近的共同父组件中

组合 VS 继承

  1. 包含关系：{props.children} 可嵌套JSX将任意组件作为子组件传递到渲染结果中（也可以自定义，不一定要children）类似于Vue的插槽
  2. 特例关系：“特殊”组件可以通过 props 定制并渲染“一般”组件：相当于给函数传递实参返回结果
  3. 组合也同样适用于以 class 形式定义的组件
  4. Props 和组合提供了清晰而安全地定制组件外观和行为的灵活方式，组件可以直接引入（import）而无需通过 extend 继承
